# Situation Report - 2026-03-28

## Highlights

1. **Flashbots introduces Flashnet: anonymous broadcast for Ethereum's mempool** — A new protocol that provides network-level sender anonymity using a clever spatial detachment strategy: clients place messages at random positions in arrays, secret-share them among servers, who sum shares locally. Unlike Tor (weak anonymity) or mix-nets (high latency), Flashnet achieves strong anonymity with low latency. TEEs enforce honest client behavior. Critical for decentralized block building — currently BuilderNet relies on trusted operators (Flashbots, beaverbuild, Nethermind); Flashnet could enable untrusted builders to participate. Targets Phase 2 of Flashbots' decentralization roadmap, requires no Ethereum consensus changes. [Link](https://writings.flashbots.net/network-anonymized-mempools)

2. **a16z: Agentic commerce will replace advertising** — Samuel Ragsdale argues LLMs/agents don't get distracted by ads, making ad-based monetization obsolete. Two open protocols are identified: **x402** (Coinbase, based on HTTP 402 status code) and **mpp** (Tempo/Stripe). Key enabler: modern LLMs (Claude 4.5+, Codex 5.2+) can discover APIs, read schemas, and use them without prior training. AgentCash provides unified merchant discovery via x402scan.com and mppscan.com. Directly relevant to Chainbound — the dashboards you track are cited as core infrastructure. [Link](https://a16zcrypto.substack.com/p/open-agentic-commerce-and-the-end)

3. **a16z: Blockchains need predictability and privacy, not just speed** — Garimidi, Neu, and Resnick argue that "eventual inclusion over seconds" is insufficient for financial applications. Single-leader protocols create two exploitable advantages: censorship power (selectively excluding txns) and information asymmetry (seeing pending txns before including own). They propose two required properties: **short-term censorship resistance** (valid txns must be included in the next block) and **transaction hiding** (contents concealed until ordering is immutable). References arXiv:2509.23984 for formal definitions. [Link](https://a16zcrypto.substack.com/p/what-blockchains-need-to-compete)

4. **Decentralized Thoughts: Strong Chain Quality** — Introduces SCQ, which guarantees stake-proportional blockspace allocation *within each block* (not just over time like classical Chain Quality). Enables "virtual lanes" — dedicated blockspace proportional to stake. Requires additional protocol rounds beyond standard BFT. Open questions remain around transaction ordering and proposer "last look." Transforms blockspace into a competitive asset market. [Link](https://decentralizedthoughts.github.io/2026-03-23-strong-chain-quality/)

5. **DPaaS: Removing relays from Ethereum PBS** — Distributes the relay function across multiple Proposer Entities running on isolated TEEs with BLS threshold cryptography. Bid processing under 5ms (vs 100-200ms for relays), post-auction latency 55-172ms (vs 880ms for MEV-Boost). Maintains compatibility with existing Ethereum consensus. Addresses the relay single-point-of-failure that has already been exploited to steal funds. [Link](https://decentralizedthoughts.github.io/2025-12-12-dpaas/)

## Blogs & Research

### MEV / Block Building / PBS

**Flashbots: "Network Anonymized Mempools" (Flashnet)** (Feb 17)
Flashnet provides sender anonymity through spatial detachment — clients place messages at random array positions, secret-share arrays among servers, who sum shares locally before the leader reconstructs output. Key tradeoff: TEEs protect liveness but not anonymity (breaking TEE degrades responsiveness only). Bandwidth overhead from cover traffic is moderate, substantially lower than mix-nets. Directly addresses selective censorship, frontrunning, and the trust assumptions in BuilderNet's current operator set. [Link](https://writings.flashbots.net/network-anonymized-mempools)

**Flashbots: "Decentralized Building: Wat Do?"** (Feb 14)
Discusses approaches to decentralized block building in Ethereum's MEV supply chain. [Link](https://writings.flashbots.net/decentralized-building-wat-do)

**Decentralized Thoughts: "DPaaS - Removing Relays in Ethereum PBS"** (Dec 2025)
Replaces centralized relays with distributed Proposer Entities on TEEs. Uses BLS threshold signatures (f+1 of n=5f-1) for validator signing, secret sharing for block distribution, and a Builder TEE component for block validation. Performance dramatically exceeds current relay architecture. Compatible with existing consensus — no protocol changes needed. [Link](https://decentralizedthoughts.github.io/2025-12-12-dpaas/)

**Decentralized Thoughts: "Strong Chain Quality"** (Mar 23)
SCQ guarantees X% stake → X% blockspace *per block*, not just probabilistically over time. Implementation requires two additional rounds: parties submit inputs and inclusion lists before block proposal. Doesn't solve ordering or "last look" — complementary mechanisms needed. Key implication: blockspace becomes a tradeable asset tied to stake, creating sustained L1 token demand. [Link](https://decentralizedthoughts.github.io/2026-03-23-strong-chain-quality/)

### Agentic Commerce

**a16z: "Open Agentic Commerce and the End of Ads"** (Mar 21)
Two open protocols: x402 (Coinbase, HTTP 402-based) and mpp (Tempo/Stripe). Modern LLMs can autonomously discover and use unfamiliar APIs. AgentCash provides unified merchant discovery. Historical parallel to open protocols (HTTP/HTML/DNS) defeating AOL's walled garden — walled-garden agent checkouts (ChatGPT, Gemini) will face the same fate. [Link](https://a16zcrypto.substack.com/p/open-agentic-commerce-and-the-end)

**a16z: "Blockchains are fast enough for finance. Now what?"** (Mar 25)
Two missing properties for DeFi competitiveness: short-term censorship resistance and transaction hiding. Market makers need immediate inclusion guarantees — without them, arbitrageurs exploit stale orders, widening spreads. Current workarounds (reputation, offchain execution via UniswapX/CoWSwap) are fragile and undermine composability. [Link](https://a16zcrypto.substack.com/p/what-blockchains-need-to-compete)

### Cryptography & Privacy

**Cryptography Engineering: "Anonymous Credentials: An Illustrated Primer"** (Mar 2)
Matthew Green covers Chaumian credentials (blind signatures, single-use with serial numbers), ZK-based credentials (NIZK proofs for complex claims like "over 25 AND US resident BUT not from NY"), and anti-cloning countermeasures (limited-use, revocation via PRF outputs, hardware binding to secure elements). Green frames anonymous credentials as "the most important topic we could be talking about as cryptographers" given proliferating age-verification legislation. [Link](https://blog.cryptographyengineering.com/2026/03/02/anonymous-credentials-an-illustrated-primer/)

**Cloudflare: "Anonymous credentials: rate-limiting bots without compromising privacy"** (Oct 2025)
Two concrete schemes: **ARC** (Anonymous Rate-limited Credentials) — parallel presentations across origins, issuance 1.35ms for 1000 presentations, 288 bytes per presentation; **ACT** (Anonymous Credit Tokens) — stateful with decremented counters on each use, enabling adaptive rate-limiting, 1867 bytes per presentation due to range proofs. Hybrid deployment: ARC for initial access from AI platform, exchanged for per-origin ACT credentials. Uses Ristretto255 curves and SHAKE128. Implementation alongside MCP integration for agent tooling. [Link](https://blog.cloudflare.com/private-rate-limiting/)

**Cloudflare: "Beyond IP lists: a registry format for bots and agents"** (Oct 2025)
Proposes open registry format where bots sign HTTP requests via `Signature-Agent` header pointing to JWKS endpoints. Registry entries are simple URL lists (like robots.txt) hostable anywhere. Extends RFC 7517 with metadata: rate expectations, purpose declarations, key validity windows. Participating: Amazon Bedrock AgentCore, Vercel, Shopify, Visa. [Link](https://blog.cloudflare.com/agent-registry/)

**Cloudflare: Post-quantum transparency** (Feb 27)
Client-side PQ encryption surged from <3% (early 2024) to >60% (Feb 2026) using X25519MLKEM768 hybrid. Origin server PQ support at ~10%, 10x increase from <1% in early 2025. Also launched Key Transparency dashboard for WhatsApp-style E2EE services (auditable append-only key logs, similar to Certificate Transparency) and ASPA routing security metrics. [Link](https://blog.cloudflare.com/radar-origin-pq-key-transparency-aspa/)

### Distributed Systems & Consensus

**Decentralized Thoughts: "Mysticeti: Revolutionizing Consensus on Sui"** (Mar 6)
Eliminates explicit block certification by using DAG structure as "virtual certificate." Multi-proposer (every block is a potential proposer slot) achieving 3-round good-case latency. Novel crash-fault masking detects benign failures after 2D+d and skips them. Performance: ~500ms latency at world scale, >100K TPS, 40% CPU reduction. Live on Sui mainnet since July 2024 across 106 validators. [Link](https://decentralizedthoughts.github.io/2026-03-06-mysticeti-revolutionizing-consensus-on-sui/)

**Decentralized Thoughts: "Synchronized Clocks and Simultaneous Agreement"** (Mar 7)
Compares Fixed View Schedules (FVS, predetermined timing) vs Variable View Schedules (VVS, adaptive). Identifies the proposer incentive problem: leaders naturally delay to accumulate txns and MEV. Proves a fundamental limitation — simultaneous agreement with time X requires X >= t+1 rounds minimum. Achieving "fixed schedules with no gaps" while exploiting good-case speedups remains an open problem. [Link](https://decentralizedthoughts.github.io/2026-03-07-simultaneous-agreement/)

## Cross-Cutting Themes

**Privacy as infrastructure, not feature.** Multiple independent signals converge: Flashnet (network anonymity for mempools), a16z's "transaction hiding" requirement, Cloudflare's anonymous credentials for agents, Matthew Green's primer on anonymous credentials. Privacy is shifting from a niche concern to a required property for both financial systems and agent infrastructure.

**The relay must die.** DPaaS (TEE-based distributed proposers) and Flashnet (anonymous broadcast for builders) both target the centralized relay as Ethereum PBS's weakest link. DPaaS replaces relay trust with threshold cryptography; Flashnet enables untrusted builder participation. These are complementary approaches that could compose.

**Agentic commerce infrastructure is real.** MPPScan shows 25.9K transactions across 638 agents. a16z frames x402 and mpp as the HTTP of agent payments. Cloudflare is building the authentication layer (agent registry + anonymous credentials). The stack is forming: identity (Cloudflare registry) → payment (x402/mpp) → discovery (AgentCash) → compute (Fly.io Sprites).

**Blockspace economics are being formalized.** Strong Chain Quality transforms blockspace from a best-effort resource into a stake-proportional entitlement. Combined with the a16z paper on censorship resistance and transaction hiding, the academic foundation for "blockspace as a market" is solidifying rapidly.

**Post-quantum transition is accelerating.** 60% client-side PQ adoption, 10% origin server support (10x YoY). The transition is happening faster than most expected.
