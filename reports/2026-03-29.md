# Situation Report - 2026-03-29

## Highlights

**1. zk-X509: ZK-Proven On-Chain Identity from Legacy PKI** — A well-engineered paper from Tokamak Network demonstrates how to bridge the 4 billion existing X.509 certificates into Web3 without on-chain PII exposure. The circuit (built on SP1 zkVM) handles full chain validation, CRL checking, nullifier generation, and selective attribute disclosure, with ZK-based CA-member hiding that expands the anonymity set to tens of millions of users. On-chain verification costs ~300K gas (Groth16), proving takes ~5 min on M1 — a once-per-certificate event. The nullifier design prevents cross-service linkability; Merkle-tree CA whitelisting hides which specific CA issued the credential. This is a concrete, deployable Sybil-resistance and compliance primitive for Ethereum — directly composable with EAS attestations for builder/relay identity or DAO governance, with no new issuance infrastructure needed.

**2. SNARE: Byzantine Consensus Tolerating 73% Adversarial Stake** — Extends the rational-player BFT model to 5f+1 nodes via a two-layer architecture: any 5f+1 protocol for predecision, plus an off-critical-path all-to-all broadcast requiring 4f+1 threshold signatures for finalization. In the "no-fork zone" (coalitions ≤ 60% of stake), safety holds unconditionally with no deposits. Deposits are only needed above 60%, and the protocol tolerates coalitions up to ~73% of stake with growing deposit requirements. The Valid-Candidacy Lemma holds unconditionally in 5f+1, removing constraints that limited prior work on MEV/front-running resistance in validator sets.

**3. The Trivy Supply Chain Attack & The SHA Pinning Illusion** — TeamPCP compromised Aqua Security's `trivy-action` and `setup-trivy` GitHub Actions, replacing the commits behind 76/77 tags with malware that scraped process memory for SSH keys, cloud credentials, Kubernetes tokens, and crypto wallets across ~10,000 CI/CD pipelines. The post-mortem argument: SHA pinning is necessary but not sufficient. A PR that updates a pinned SHA to attacker-controlled code is invisible to human review — SHA hashes are opaque. The real solution stack requires automated SHA-to-release-version mapping (Dependabot/zizmor), Sigstore/cosign provenance attestations (SLSA), and minimal GITHUB_TOKEN scopes. Directly relevant to any blockchain infrastructure with CI/CD pipelines.

**4. AetherWeave: Stake-Weighted Sybil-Resistant P2P Peer Discovery** — Proposes replacing zero-cost network identities with stake-backed identities in P2P overlay protocols. Sybil attacks require locking capital proportional to fake identity count; misbehavior triggers on-chain slashing via publicly verifiable proofs. Critically, stake ownership is proven via ZK proofs — nodes cannot be deanonymized via their deposit. The protocol guarantees that eclipsed honest nodes can detect the attack, preventing silent partitioning. Validated with a Prysm fork, signaling real deployment intent for Ethereum's discv5/libp2p layer.

**5. x402 Protocol: 45K Transactions, $16.7K Volume in 24h** — Agentic commerce on x402 continues growing. Virtuals Protocol AI agents on Base (acp-x402.virtuals.io) dominate at 11.5K txns and $8,980 volume — over half of total dollar volume with 419 unique buyers. **How Not to Mandate Device-Based Age Assurance** (Privacy) — [Link](https://educatedguesswork.org/posts/device-based-age-assurance/)
EKR (IETF TLS WG Chair) dissects state legislation (CA AB 1043, TX SB 2420, UT SB 142, and several others) mandating OS-level age-range APIs. The technical critique: (1) an unrestricted age-range API creates a birthday inference oracle — repeated queries can pinpoint a user's exact birthdate from discrete category transitions; (2) legislation requiring all apps to query the API generates mandatory demographic data leakage to irrelevant software (calculators, flashlights); (3) jurisdiction detection forces IP-based geolocation, creating location tracking infrastructure for previously privacy-neutral apps; (4) device filtering mandates are technically infeasible against encrypted DNS and modern SPAs. A case study in how coarse privacy-preserving signals (age ranges instead of birthdates) introduce new attack surfaces when the query model isn't constrained.

**The Comforting Lie of SHA Pinning** (Security) — [Link](https://www.vaines.org/posts/2026-03-24-the-comforting-lie-of-sha-pinning/)
Written in the wake of the Trivy compromise (GHSA-69fq-xp46-6x23). Core argument: SHA pinning defeats tag-poisoning attacks but relocates trust to an opaque namespace humans cannot audit. A PR updating a pinned SHA to attacker-controlled code looks identical to a legitimate dependency update. Better stack: automated SHA-to-release mapping (zizmor, Dependabot), Sigstore/cosign attestations for SLSA provenance, in-house action mirroring, and minimal GITHUB_TOKEN scopes. The transitive dependency problem is unaddressed by SHA pinning alone — runtime downloads within an action execute with full trust regardless of how the entrypoint is pinned.

**Register Spill: Joy & Curiosity #80** (Agentic Systems) — [Link](https://registerspill.thorstenball.com/p/joy-and-curiosity-80)
Thorsten Ball's knowledge-location framework: agent success depends on whether required knowledge exists in (1) the prompt, (2) the codebase, or (3) model training data. Agents fail when knowledge is absent from all three — not due to capability limits, but ambiguity in the solution space. Practical implication: prompt engineering is knowledge injection; well-typed codebases and consistent conventions reduce prompt burden. John Regehr's "zero-degree-of-freedom" prescription (use executable oracles — tests, type checkers — to eliminate valid-solution ambiguity) is cited as a concrete engineering response. Also notable: Cheng Lou's *Pretext* (TypeScript text-measurement without CSS/DOM) as an example of Claude-Code-assisted novel algorithm development over weeks of iteration.

## Academic Papers

**zk-X509: Privacy-Preserving On-Chain Identity from Legacy PKI via Zero-Knowledge Proofs** — [arXiv:2603.25190](https://arxiv.org/abs/2603.25190)
SP1 zkVM circuit performs 6 checks: chain validation, temporal validity, key ownership, CRL checking, registrant binding, and configurable nullifier generation. Nullifiers are derived from deterministic signatures over domain-separated strings (including contract address and chain ID), preventing cross-service linkability. CA membership is proven via Merkle tree without revealing which CA issued the credential, pooling anonymity across whitelisted CA populations (~67M with NPKI + Estonian + German eID). Proving: ~5 min M1 for RSA-2048; ~11.8M SP1 cycles for P-256. On-chain: ~300K gas Groth16. Deployable within 3–6 months with no new issuance infrastructure.

**AetherWeave: Sybil-Resistant Robust Peer Discovery with Stake** — [arXiv:2603.23793](https://arxiv.org/abs/2603.23793)
Stake-backed P2P discovery with per-node communication scaling O(s√n). Rate-limiting via cryptographic commitments; violations produce on-chain-verifiable misbehavior proofs triggering automatic slashing. Privacy: stake ownership proven via ZK proofs, preventing deanonymization via deposit linkability. Safety guarantee: honest overlay stays connected, or isolated nodes detectably know they are eclipsed. Validated on a Prysm fork.

**Rafture: Erasure-Coded Raft with Post-Dissemination Pruning** — [arXiv:2603.24761](https://arxiv.org/abs/2603.24761)
Integrates erasure coding into Raft using adaptive fragment assignment at dissemination time (leader sends ⌈(F+1)/t⌉ fragments per responsive node). Key novelty: post-dissemination pruning — after commit, nodes asynchronously discard redundant fragments as the leader piggybacks updated responsive-quorum estimates in heartbeats. No re-encoding or extra coordination required. Achieves 4-hop latency ~95% of the time vs. ~60% for CRaft/HRaft under instability, and up to 50% storage reduction vs. FlexRaft after partition recovery. Uniform reconstruction eliminates per-entry metadata complexity.

**SNARE: Byzantine Consensus for Rational Players in the 5f+1 Model** — [arXiv:2603.23458](https://arxiv.org/abs/2603.23458)
Two-layer architecture: 5f+1 predecision protocol + off-critical-path BFTCR finalization requiring 4f+1 threshold signatures. No-fork zone (coalitions ≤ 3f, ~60%) requires zero deposits. Deposits scale sublinearly between 3f–4f; above 4f deposits grow with coalition size up to ~73% tolerance. Valid-Candidacy Lemma holds unconditionally in 5f+1, removing prior constraints. Proves impossibility of deposit-free safety above the 60% threshold, establishing a fundamental lower bound on mechanism complexity.

**TEE for Solving the Replication Crisis in Academia** — [arXiv:2603.24878](https://arxiv.org/abs/2603.24878)
Proposes Intel TDX-backed enclaves for academic reproducibility: researchers submit code, data providers submit encrypted datasets, TDX runs analysis and produces unforgeable attestation certificates. Journals validate attestation artifacts instead of re-running code. Cost: $1.35–$1.80 per replication package. The attestation mechanism is architecturally identical to TEE-based block building — the paper demonstrates that TDX's cost and usability profile now supports non-specialist users, which is relevant to arguments about TEE viability in trust-minimized PBS contexts.

**Amplified Patch-Level Differential Privacy for Free via Random Cropping** — [arXiv:2603.24695](https://arxiv.org/abs/2603.24695)
Formalizes random cropping (universal data augmentation) as a DP amplification mechanism with no additional noise or architectural changes. Effective sampling rate: γ_eff = γ_wo × γ_crop (minibatch × crop inclusion probability). Results on semantic segmentation: up to 330% mIoU improvement at ε ≈ 5 for PSPNet on Cityscapes. Directly applicable to medical imaging and autonomous driving datasets with spatially localized sensitive content.

## New Releases

**bigoish 0.x** (Rust) — [docs.rs/bigoish](https://docs.rs/bigoish/)
Rust crate for empirically testing algorithmic complexity within `cargo test`. Uses CPU instruction counts (Linux perf counters) for noise-resistant measurement; falls back to thread CPU time on other platforms. Supports O(1) through O(n³) and custom models via operator-overloaded composition (`N * Log(N)`). Useful for catching complexity regressions (O(n log n) → O(n²)) that benchmarks and correctness tests miss. Terminal-rendered diagnostic graph on failure.
