## Highlights

**1. DeFi Composability Breaks AML by Design** — A formal paper from Yixin Cao et al. proves that existing blockchain AML frameworks are structurally incapable of attribution-completeness in DeFi. The argument: DeFi routinely separates Principal (economic intent), Execution (transaction submitter), and Beneficiary (recipient) across different addresses — decoupled by composable protocols, not deliberate obfuscation. MEV infrastructure (searcher bundles, intent-based settlement, relayers) is the primary structural vehicle for this separation at scale. A second mechanism — state-mediated value migration via AMM reserve changes, lending rates, and perpetual funding accumulators — allows value to flow between parties with zero token transfer events visible to AML monitors. The paper argues that heuristic transfer-graph tracing has reached its observational limit and execution-semantics-based attribution is the only viable path forward. Directly relevant to PBS, MEV research, and compliance-by-design protocol features.

Applied to 6 live production DeFi projects, it found 12 high-severity and 10 medium-severity previously unknown vulnerabilities. The key advance: anchoring detection in DeFi-specific business-logic patterns extracted from real past audits, not abstract code patterns — bridging the gap between automated tools and human auditor reasoning.

**4. Cloudflare Reads Your React State** — A reverse-engineering post (753 HN points) decrypts Cloudflare Turnstile's fingerprinting mechanism as deployed on ChatGPT. 55 properties are collected: WebGL GPU strings, hardware concurrency, screen resolution, font rendering metrics, behavioral biometrics (keystrokes, mouse velocity, scroll), SHA-256 hashcash puzzles, and — the novel part — three React-specific state checks (`__reactRouterContext`, `loaderData`, `clientBootstrap`) that gate user input until the React app has fully hydrated. The 28,000-character encrypted bytecode uses a two-layer XOR scheme (outer key from the prepare-request token, inner key from a float literal embedded in the payload) — obfuscation, not cryptography. Any party with the XOR key can read all collected data. The privacy guarantee is entirely policy-based.

**5. Bitcoin Smart Accounts: Native Bitcoin DeFi via TEE + Emulated Covenants** — Protocol enabling native Bitcoin collateral for DeFi without bridging or wrapping. Pre-signed PSBTs with Taproot scripts lock funds into a 4-state state machine before deposit; even if the TEE (AWS Nitro Enclave) is compromised post-setup, pre-signed transactions can only send Bitcoin to the depositor or Token Operator. AWS Nitro PCR attestation ensures dispute logic runs unmodified. Trust requirement: 1-of-(k+1) — any single honest Arbitration Oracle or an honest Token Operator suffices. Outperforms BitVM (no multi-round interactive proofs, standard DeFi liquidation), multisig bridges (self-custody), and WBTC (non-custodial).

**Radicle: Replay Attack Vulnerability in Signed References** (Lobsters, 2 pts) — [Link](https://radicle.xyz/2026/03/30/disclosure-of-vulnerability-in-signed-references)
The signed reference scheme covered only the content of the `refs` file, with no timestamp, nonce, or parent-commit binding. An attacker could replay any historical `(refs, signature)` pair as a child of the victim's latest commit — cryptographically valid, indistinguishable from a legitimate state change. Enabled rolling a repo back to empty state or re-exposing patched vulnerabilities. Fixed in Radicle 1.7.0 by including `refs/rad/sigrefs-parent` in the signed payload, binding each signed state to its position in history. Discovered by Felix Bargfeldt, reported Feb 14 2026. 199 repositories showed duplication clusters potentially indicative of exploitation; no confirmed active exploitation.

## Academic Papers

**PEB Separation and State Migration: DeFi AML Evasion** — [arXiv:2603.26290](https://arxiv.org/abs/2603.26290)
Formal proof that DeFi composability makes transfer-layer AML attribution structurally incomplete. Two mechanisms: (1) PEB separation — DeFi protocols decouple the Principal (intent), Execution (transaction), and Beneficiary across addresses routinely; MEV searcher bundles and intent-based settlement are the primary operational vehicles. (2) State-mediated value migration — AMM reserve ratio changes, lending utilization shifts, and perpetual funding accumulators transfer economic value with zero ERC-20 Transfer events. Proposes execution-semantics-based attribution (parsing smart contract execution traces and state invariants) as the only viable alternative. Significant implications for OFAC/FATF/MiCA compliance frameworks and MEV supply chain design.

**Knowdit: Agentic Smart Contract Vulnerability Detection** — [arXiv:2603.26270](https://arxiv.org/abs/2603.26270)
Knowledge-graph-grounded multi-agent system that mines past audit reports to identify recurring DeFi vulnerability patterns (AMM invariant mechanics, vault share accounting, ERC-4626 inflation, price oracle manipulation) and applies them via iterative fuzz-harness generation + reflection loops. 100% high-severity detection, 77% medium-severity, 2 false positives on 75 ground-truth vulns; 22 new vulns in 6 live production protocols. Substantially outperforms GPTScan, PromFuzz, and PropertyGPT.

**Bitcoin Smart Accounts: Trust-Minimized Native Bitcoin DeFi via TEE** — [arXiv:2603.26293](https://arxiv.org/abs/2603.26293)
Pre-signed PSBT + Taproot 4-state machine emulates covenants on Bitcoin. Arbitration Oracles run in AWS Nitro Enclaves with PCR attestation; pre-setup signing ceremony ensures post-compromise funds can only go to depositor or TO. Trust model: 1-of-(k+1). Addresses taproot address uniqueness via BIP-341 NUMS internal key tweaking. Handles fee variability via CPFP + anchor outputs. Detailed comparison to BitVM (no interactive fraud proofs needed), Babylon (no bespoke liquidation), WBTC (non-custodial).

**Reentrancy Detection in the Age of LLMs** — [arXiv:2603.26497](https://arxiv.org/abs/2603.26497)
31-tool evaluation across static analysis, ML models, and LLMs (including GPT-4 and Claude) on 432 real contracts + 143 minimal RSD contracts isolating individual reentrancy patterns. LLMs achieve 0.96 F1 on the aggregated benchmark vs. 0.87 for best traditional tools; 0.82 vs. 0.76 on the harder RSD. Traditional tools degrade sharply on modern Solidity 0.8+ due to maintenance gap; they miss read-only reentrancy (no state-write, value change via delegated state reads — core Curve-exploit vector). Recommendation: LLM + static analysis in combination for best coverage.

**Hermes' Seal: ZK for Autonomous Vehicle Communications** — [arXiv:2603.26343](https://arxiv.org/abs/2603.26343)
Groth16-based zk-SNARK framework for V2V/V2X safety attestation. Proves bounded safety claims (RSS-compliant stopping distance, model Precision/Recall thresholds) without revealing sensor data, model weights, or GPS. 88ms proof / 11ms verification on GPU. Key design: private key never enters zkVM; OS keychain (Secure Enclave, TPM) signs challenges, only signatures enter the circuit. Domain-separated commitment via Poseidon hash with 128-bit blinding. Trusted setup (Groth16) managed by Enforcement Authority — the main deployment constraint; requires re-ceremony when safety policy changes.

**Linearly Homomorphic Ring Signatures over Lattices** — [arXiv:2507.02281](https://arxiv.org/abs/2507.02281)
First provably secure lattice-based LHRS: ring anonymity + linear homomorphism + complete security proof (fixing CK17/CK18's unproven scheme). SIS-only hardness assumption. Combines signed message vectors via linear combination of signature vectors; norm bound maintained up to k₀=k/2 operations. Anonymity is information-theoretic (unconditional). Ring size constraint: O(log n) — a significant practical limitation imposed by the reduction's success probability. Applications: anonymous e-voting, confidential transaction aggregation, outsourced computation.

**Cryptanalysis of a PIR Scheme based on Linear Codes over Rings** — [arXiv:2603.26409](https://arxiv.org/abs/2603.26409)
Polynomial-time attack on the Bodur-Martínez-Moro-Ruano (2025) single-server PIR scheme. Attack: dimension-based distinguisher over ℤ_{p_i}-components — the true query index produces strictly lower ℤ_p-dimension in the response residual matrix than all other candidates. Root cause: the scheme's definition of "non-free" code membership deviates from the standard coding-theoretic definition, making the "protective" non-free structure transparent to the dimensional distinguisher. Complexity: polynomial (Gaussian elimination dominated). Implication: ring-based codes provide no security advantage over fields against dimensional distinguishers; LWE-based PIR remains the post-quantum standard.

**CryptOracle: Modular FHE Benchmarking Framework** — [arXiv:2510.03565](https://arxiv.org/abs/2510.03565)
Benchmarking platform for OpenFHE covering application-level workloads, operation microbenchmarks, and NTT primitives. Hardware profiler captures L1/L2/L3 cache misses, memory bandwidth, pipeline stalls, power consumption. Predictive model: -7% to +8.4% runtime error, -9.7% to +15.7% energy error. Key finding: **memory bandwidth is the dominant bottleneck** for CKKS — polynomial multiplication via NTT saturates memory channels before CPU throughput binds. Bootstrapping dominates deep-circuit costs. For FHE accelerator design, the leverage point is HBM memory bandwidth, not additional compute units.

**Combinatorial Privacy: Private Multi-Party Bitstream Grand Sum** — [arXiv:2603.22808](https://arxiv.org/abs/2603.22808)
MPC protocol encoding private bits as 2×2 block-diagonal permutation matrices in the Birkhoff polytope. Aggregate sum recoverable via bilinear form; individual bits hidden by masking with random doubly stochastic matrices (convex combinations of decoy permutations). Server security: information-theoretic (zero statistical distance). Aggregator security: recovering individual bits requires computing the permanent of a matrix — #P-hard. Supports multi-statistic extraction without client resubmission (different extraction vectors applied to same transmission). Open problem: achieving both #P-hardness and non-vacuous DP simultaneously in one configuration.

**Hidden Elo: Private Matchmaking via FHE** — [arXiv:2603.26407](https://arxiv.org/abs/2603.26407)
FHE-based Elo rating system where no party — including the matchmaking server — learns individual player ratings. Ratings are stored and updated as ciphertexts; matchmaking quality function is evaluated homomorphically. Relevant to privacy-preserving mechanism design and any system requiring confidential comparative scoring (dark pool order matching, sealed-bid auctions, anonymous reputation systems).

**TRIP-RAG: Dynamic Anonymization for Privacy-Preserving RAG** — [arXiv:2603.26074](https://arxiv.org/abs/2603.26074)
Context-aware selective anonymization framework for Retrieval-Augmented Generation. Rather than uniformly stripping PII, it assesses context to anonymize only sensitive spans, preserving utility. Relevant to privacy-preserving AI inference and any pipeline passing user data to third-party LLM APIs.
