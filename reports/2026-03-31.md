# Situation Report - 2026-03-31

## Highlights

**1. Ordering Power is Sanctioning Power: PBS Enables Sanction Evasion at Scale** — A formal paper (Di Wu et al.) measures MEV-driven sanction evasion across 8 years of Ethereum data and proves that on-chain enforcement is structurally defeated by block ordering economics. Empirical findings: 7.3% of 1,990 sanctioned USDT addresses ($1.41B frozen total) and **18.7% of 123 USDC addresses** ($108.25M frozen total) were fully drained before the freeze landed. The mechanism escalates in four stages: gas-price wars → private order flow (Flashbots) → builder-level ordering manipulation → **direct proposer bribery** (ETH transferred to fee-recipient addresses, 122+ documented instances). The game-theoretic result is stark: in the Tullock contest model, stricter regulation (higher penalties Ψ) mechanically inflates the bribe economy without bound — the MEV tax extracted by proposers grows as T* = (r/(r+2))V, diverging as Ψ/V → ∞. PBS architecture amplifies all four stages by providing dedicated infrastructure for private routing and bribery. Mitigations require protocol-level changes — TEE-assisted ordering or trusted sequencing — not just policy.

**2. BadAML: ACPI Bytecode Breaks Confidential VMs** — A Black Hat Europe 2024 / ACM CCS 2025 paper (Contrast confidential computing team) demonstrates that ACPI AML bytecode achieves arbitrary code execution inside AMD SEV-SNP and Intel TDX confidential VMs. The attack vector: the Linux kernel's ACPI subsystem interprets host-supplied SSDT tables at kernel privilege, with `OperationRegion` primitives that allow arbitrary physical memory read/write. The PoC patches the initrd byte-by-byte (loaded at deterministic addresses by guest OVMF) and spawns a root shell on the serial console — no anomalous syscalls, no CVE, no kernel faults. The gap: CPU memory isolation protects encrypted pages from the hypervisor at the VMM level, but AML executes *inside* the guest above that boundary. Affects "most CVM-based confidential computing products." Mitigation: page-table-aware AML sandbox that checks whether target pages are private before allowing AML access. Direct implication: TEE-based block builders (SUAVE, TDX-backed MEV infrastructure) are vulnerable if they accept host-controlled ACPI configuration.

**3. Axios NPM Supply Chain Attack** — Axios (top-10 npm package, billions of downloads) had versions 1.14.1 and 0.30.4 poisoned for ~2-3 hours. The attack injected a phantom dependency (`plain-crypto-js@4.2.1`, never imported in source) whose `postinstall` hook downloaded platform-specific RATs (macOS Secure Enclave path, Windows VBScript dropper, Linux Python RAT), contacted C2 at `sfrclak.com:8000` within 2 seconds, then self-destructed — leaving `npm install` exit code 0 and no trace. Pre-staged a clean decoy `plain-crypto-js@4.2.0` 18 hours prior to evade zero-history detection. Forensic signal: legitimate axios releases use npm OIDC Trusted Publisher (CI-bound cryptographic release signing); the malicious versions were manually published with a stolen long-lived token. Runtime monitoring (Harden-Runner) was the only mechanism that caught live exploitation in CI.

**4. Web PKI Redesigned for Post-Quantum: Merkle Tree Certificates** — Google's IETF proposal (draft-ietf-plants-merkle-tree-certs, PLANTS WG) addresses the 10× TLS handshake size explosion that post-quantum signatures would cause. Landmark MTCs replace per-certificate CA signatures and CT SCTs with a Merkle inclusion proof against a batch tree root published to CT logs — the CA signs the tree root once for thousands of certificates. Proof sizes remain constant regardless of PQ signature size. Trade-offs: browsers must fetch CT log state out-of-band (new liveness dependency); servers must support three certificate types simultaneously (standard X.509, standalone MTC, landmark MTC); TLS negotiation complexity increases substantially. Two-year runway: Cloudflare collaboration in 2026, bootstrap target 2027.

**5. Filecoin Finality: 30× Improvement via Dynamic Analysis** — The Finality Calculator paper (Goren & Soares) shows the 2^{-30} security target is achievable in ~30 epochs (~15 min) under empirically calibrated network conditions — vs. the current hardcoded 900-epoch (~7.5 hour) fixed threshold. The model is dynamic: it ingests live chain data (miner power distributions, observed block rates) rather than worst-case static assumptions, and computes the minimum confirmation depth at each round. The 900-epoch threshold is a conservative guarantee for worst-case parameters that rarely occur. Implication: cross-chain bridges and settlement protocols using the Filecoin fixed threshold have 30× latency slack that principled dynamic finality oracles could eliminate.

## Academic Papers

**Ordering Power is Sanctioning Power: Sanction Evasion-MEV and the Limits of On-Chain Enforcement** — [arXiv:2603.27739](https://arxiv.org/abs/2603.27739)
Eight-year empirical study of stablecoin freezes on Ethereum. Documents four-stage escalation from gas auctions to direct proposer bribery. Tullock contest model proves compliant issuers cannot rationally opt out of MEV bidding (never-bid is strictly dominated), and that penalty-severity increases are counter-productive (higher Ψ = larger bribe market). Fixed costs concentrate evasion in specialized MEV bots. Calls for protocol-level mitigations (TEE-assisted ordering, trusted sequencing) as the only viable path.

**HFIPay: Privacy-Preserving Cross-Chain Payments to Human-Friendly Identifiers** — [arXiv:2603.26970](https://arxiv.org/abs/2603.26970)
ZK-ACE claim authorization: at claim time, recipient proves knowledge of the epoch-scoped handle that derives the on-chain blinded binding `ρ = H("hfipay:bind" ∥ u_{B,e} ∥ intentId)` — relay cannot redirect post-funding. Per-intent fresh `intentId` ensures pre-claim unlinkability (multiple payments to same address are indistinguishable). Verified-quote mode adds binding-issuer attestation to close pre-funding substitution vector. Cross-chain via n-Vm bridge with identical ZK relations. Contrasts with ENS (public mapping, fully linkable history) and ERC-5564 stealth addresses (continuous receiver scanning required).

**Silent Guardians: Two-Server MPC Decision Tree Evaluation Without Inter-Server Communication** — [arXiv:2603.28143](https://arxiv.org/abs/2603.28143)
First two-server ODTE protocol with zero server-to-server (S2S) communication and malicious-server security. SIC (Secure Integer Comparison) reformulates bit comparison as a RMS program evaluable locally via Paillier-based HSS — no synchronization between servers needed. Gate count reduced from 6t−4 to 4t−2 multiplications for t-bit comparison. Authentication tags (MAC key held by client) detect any server result tampering. 20× communication reduction and 2.7× faster than prior state-of-the-art. Non-colluding assumption justified operationally by deploying on independent cloud providers.

**Attacks on Sparse LWE and Sparse LPN with New Sample-Time Tradeoffs** — [arXiv:2603.27190](https://arxiv.org/abs/2603.27190)
Kikuchi method generalized from ℤ₂ to ℤ_q, attacking sparse LWE/LPN with higher moduli. Spectral method: time `Õ(C(n,l)·q^l)`, samples `(n/l)^k·log^c(n)`. Cover/walk method: ~quadratic speedup for `l ≫ k`, requires prime q. Confirms the Jain-Lin-Saha (2024) hardness conjecture boundary is tight — construction supporting constant-overhead linearly homomorphic encryption is essentially at the limit. Unaffected: CRYSTALS-Kyber, NTRU, and standard (dense-secret) LWE schemes.

**The Finality Calculator: Filecoin's Probabilistic Finality** — [arXiv:2603.01307](https://arxiv.org/abs/2603.01307)
Dynamic finality model calibrated from live chain data vs. worst-case static assumptions. 2^{-30} error probability at ~30 epochs under typical conditions, vs. current 900-epoch fixed threshold. Dynamic calculator emits a time-varying depth recommendation adaptive to observed network state. Framework generalizes to other DAG-structured blockchains. Primary implication: cross-chain bridges can reduce finality latency by 30× by querying a dynamic oracle rather than hardcoding conservative epoch counts.

**A Density-Delay Law for Stable Distributed Systems** — [arXiv:2603.26758](https://arxiv.org/abs/2603.26758)
Formal proof that fork-depth stability requires λΔ = O(1) (proposal rate × propagation delay). Fork depth modeled as birth-death process; stability requires negative drift (P_coll < q). Inverse-scaling law: as participant count N grows, per-participant proposal rate must scale as O(1/N) to maintain aggregate λ = O(1). Bitcoin's difficulty adjustment reinterpreted as a decentralized density regulator — not hash-rate normalization. Implication: compact block relay (reducing Δ) directly relaxes the throughput constraint without sacrificing fork-depth stability.

**ORACAL: Heterogeneous Graph + RAG Smart Contract Vulnerability Detection** — [arXiv:2603.28128](https://arxiv.org/abs/2603.28128)
Fuses CFG (EtherSolve bytecode), DFG (Solidity AST), and call graph (Slither) into a unified heterogeneous graph. RAG pipeline (BAAI/bge-large-en-v1.5 + cross-encoder reranking + Gemini Flash) enriches top-50 critical nodes with structured security context. Causal attention with dual-branch architecture (causal vs. spurious features) suppresses adversarial perturbation to 3% Attack Success Rate vs. 10.9–18.7% for competitors. 91.28% Macro F1 on SoliAudit (+8.65pp vs. next best MANDO-HGT). Detects reentrancy, access control, arithmetic, low-level calls, DoS, front-running, time manipulation. Compared to Knowdit: ORACAL is faster and adversarially robust; Knowdit is more generalizable to novel business-logic vulnerabilities via KG traversal.

**Efficient Encrypted Inference in Spiking Neural Networks with TFHE** — [arXiv:2603.26781](https://arxiv.org/abs/2603.26781)
FHE inference on encrypted data using TFHE bootstrapping for spiking neural networks (SNNs). <3% accuracy loss vs. plaintext, sub-second inference on MNIST. SNNs' sparse binary activations reduce FHE gate count vs. standard DNNs, making them naturally FHE-friendly.

**Information-Theoretic Solutions for Seedless QRNG Bootstrapping and Hybrid PQC-QKD** — [arXiv:2603.26907](https://arxiv.org/abs/2603.26907)
Universal hash functions as strong extractors solve the QRNG entropy bootstrapping problem (how to extract uniform randomness before a QRNG is trusted). Extends to hybrid post-quantum + QKD key combination with quantifiable security bounds. Relevant to post-quantum cryptographic infrastructure.
